# 贪吃蛇小游戏项目方案

## 1. 项目目标
使用原生 HTML、CSS、JavaScript 开发一个可直接在浏览器中运行的贪吃蛇小游戏。

目标要求：
- 打开页面即可开始游戏，无需安装额外依赖。
- 具备完整的基础玩法：蛇移动、吃食物、变长、得分、撞墙/撞自己后游戏结束。
- 页面简洁直观，适合学习和演示前端基础交互。

## 2. 功能规划

### 核心功能
1. **游戏地图**
   - 使用固定大小的网格作为游戏区域。
   - 蛇和食物都基于网格坐标绘制。

2. **蛇的移动**
   - 蛇按固定时间间隔自动移动。
   - 玩家通过键盘方向键控制移动方向。
   - 禁止直接反向移动，避免逻辑错误。

3. **食物生成与得分**
   - 地图上随机生成食物。
   - 蛇吃到食物后身体增长。
   - 分数实时更新。

4. **碰撞检测**
   - 撞墙后游戏结束。
   - 撞到自己身体后游戏结束。

5. **游戏控制**
   - 提供“开始游戏 / 重新开始”按钮。
   - 游戏结束时显示提示信息。

### 可选优化（如你后续同意可再加）
- 暂停/继续功能
- 难度切换
- 最高分记录（localStorage）
- 移动端按钮控制
- 动画和音效

## 3. 页面结构规划
计划生成以下文件，全部放在 `demo_1/` 目录下：

1. `plan.md`
   - 项目方案说明文档。

2. `index.html`
   - 页面主体结构。
   - 包含标题、分数区、游戏区域、按钮区。

3. `style.css`
   - 页面布局与游戏样式。
   - 包括容器、棋盘、蛇、食物、按钮、提示文本等样式。

4. `script.js`
   - 游戏核心逻辑。
   - 包括初始化、渲染、移动、碰撞检测、食物生成、重新开始等功能。

## 4. 技术栈
- **HTML5**：搭建页面结构。
- **CSS3**：实现页面布局与视觉样式。
- **原生 JavaScript (ES6)**：实现游戏逻辑与交互。

## 5. 实现方案

### 方案选择
本项目优先采用 **DOM 网格渲染方案**：
- 页面中生成固定数量的小方格。
- 通过给对应格子添加类名来表示蛇头、蛇身和食物。

### 选择原因
- 更适合学习 HTML/CSS/JS 基础。
- 代码直观，便于理解坐标与渲染关系。
- 对于小型练手项目足够稳定。

> 如果你更希望使用 `canvas` 版本，我也可以在你确认后改成 canvas 实现。

## 6. 开发步骤
1. 创建基础目录和文件。
2. 完成 `index.html` 页面结构。
3. 完成 `style.css` 基础样式和棋盘布局。
4. 完成 `script.js` 核心逻辑：
   - 初始化棋盘
   - 初始化蛇
   - 键盘控制
   - 自动移动
   - 食物生成
   - 碰撞检测
   - 分数更新
   - 游戏结束与重开
5. 在浏览器中验证基础功能是否正常。
